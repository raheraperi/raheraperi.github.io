Technical Blog - Grids and Responsive Web Design

March 2018

Why should you care about grid-based design?

It’s a universal design method of laying out your webpage into a series of hypothetical rows and columns to form a grid. We use this grid to line up our design elements in a way that both makes sense for the user and is aesthetically appealing.
Another Important function of grids is that they create a responsive environment wherein when a page is resized or viewed on devices of different resolutions they don’t move or change - this creates consistency.
Typically, grids consist of columns, rows and the gaps between rows and columns (gutters). This creates a consistent experience, as a grid system enables the same size ratio between all your different design elements. Your grid system may also be a framework — either third party such as Bootstrap or created by you just for your project — that you use to enforce the grid via CSS.

What's all the hype about responsive web pages?

Responsive web page design is super important as it can significantly impact on the user experience. We want the user to have the best experience possible on whatever device thy might be using, whether it be a tablet, phone or laptop - meaning that we must design our web pages to adapt to different resolution sizes. Interestingly, a 2015 study by Research New Zealand found that 70 per cent of participants had access to a smartphone, 72 per cent had access to a laptop and 57 per cent had access to a PC. Of the smartphone owners, one in two preferred using their smartphone for browsing the internet. We must take user accessibility into account when designing adaptive web pages.

What the heck is semantic structure?

The phrase “it’s all just semantics” shrugs off the fact that it’s all just words and words don’t matter. Semantic markup, however, matters a lot!
Using semantic markup is essentially a way of structuring HTML to reinforce the meaning of the content rather than what it looks like. This reflects the the fact that presentation and content are separate, i.e. HTML is used for structure and CSS for web design.
When structuring our pages we need to observe the intuitive hierarchy of our markup. What I mean by this is that, much like writing an essay, we would start with our title or heading tag <h1>, including subheadings where necessary <h2>, <h3>and writing paragraphs <p> in the body of our essay.
Semantic markup means that HTML tags are never chosen based on the way they appear in a web browser — they’re chosen based on the importance and structure of the content.

Interesting stuff!

Grid structures seem like the bread and butter of responsive web design. I had a lot of breakthrough moments during the Udacity tutorial on grid structures - understanding how to create one but also how to structure your design elements within your grid.
I can’t wait to start using Bootsrap more for my grid frameworks. Seems like a handy tool for responsive web design, with predefined classes for easy layout as well as the option to use your own CSS stylesheet alongside it.
