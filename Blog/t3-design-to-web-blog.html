Responsiveness is key

A responsive site looks good on all devices, whether it be a smartphone, tablet or desktop. This is fundamental to a great user experience, as if our design elements remained static or unchanged for a web page designed for desktop, our friends who predominantly use their phones to surf the web will have a pretty average experience.

Mobile-first, everything else later!

It just makes sense! Mobile-first design means designing your smallest screen first and working your way up from there.

If your site is good on a mobile device, it translates better to all devices. More important, though, is that the mobile-first approach is also a content-first approach. Mobile has the most limitations, screen size and bandwidth to name a few, and so designing within these parameters force you to prioritize content ruthlessly.

The mobile-first approach leads to a design that’s more content-focused, and therefore user-focused. The heart of the site is content — that’s what the users are there for.

Frameworks are a godsend

A front-end framework is a package containing pre-written code in files and folders. Basically you have everything you need in this package - a grid to organise your design elements, defined font styles and sizing, and website components such as panels, buttons and navigation bars.

I used Bootstrap to re-build my site and it was pretty straightforward and allowed me to implement a couple of cool things like my dropdown nav bar (my first Javascript - milestone!). I also found it super simple to understand the grid structure and how to lay things out on my page. Not to mention that it was a cinch to make the site responsive!

Pros:

  • Save time!
  • Basics sorted
  • Straightforward grid structure
  • Add fancy stuff easily
  • Responsive design easily implemented

Cons:

  • Customisation restrictions (although you can add your own CSS stylesheet to help with this)

Wireframe your way to fame

Wireframing is a way to design a website at the structural level. A wireframe is used to lay out content and functionality on a page which thinks about user needs and user journeys. Wireframes are used early in the development process to define the basic structure of a page before visual design and content is added.

A wireframe is a layout of a web page that demonstrates what interface elements will exist on key pages. It is a critical part of the interaction design process.

The aim of a wireframe is to provide a visual understanding of a page early in a project to get stakeholder and project team approval before the creative phase gets under way.

Tricky stuff

I used Bootstrap to rebuild my blog which made it super easy to use a responsive grid framework. I found it difficult to size my images correctly for my mobile layout, they’re also not quite aligned correctly.
